# Практическая работа №19: PostgreSQL + Sequelize

## Описание

REST API для управления пользователями с использованием:
- **PostgreSQL** — реляционная база данных
- **Sequelize** — ORM (Object-Relational Mapping)
- **Express** — веб-фреймворк

## Сущность "Пользователь"

| Поле | Тип | Описание |
|------|-----|----------|
| id | Integer | Уникальный идентификатор |
| first_name | String | Имя |
| last_name | String | Фамилия |
| age | Integer | Возраст |
| created_at | Timestamp | Время создания (Unix timestamp) |
| updated_at | Timestamp | Время обновления (Unix timestamp) |

## API Endpoints

| Метод | Маршрут | Описание |
|-------|---------|----------|
| POST | `/api/users` | Создание пользователя |
| GET | `/api/users` | Получение всех пользователей |
| GET | `/api/users/:id` | Получение пользователя по ID |
| PATCH | `/api/users/:id` | Обновление пользователя |
| DELETE | `/api/users/:id` | Удаление пользователя |

# Практическая работа №21: Redis + Кэширование

## Описание

Добавление кэширования в приложение с ролевой моделью (RBAC) с использованием **Redis**.

Кэширование позволяет:
- уменьшить нагрузку на сервер и базу данных
- ускорить ответ клиенту
- повысить масштабируемость приложения

## Технологии

| Технология | Назначение |
|------------|------------|
| **Redis** | In-memory хранилище данных типа key-value |
| **Express** | Веб-фреймворк |
| **JWT** | Аутентификация и авторизация |

## Кэшируемые маршруты

| Маршрут | Метод | Время кэша | Описание |
|---------|-------|------------|----------|
| `/api/users` | GET | 1 минута | Получение списка пользователей |
| `/api/users/:id` | GET | 1 минута | Получение пользователя по ID |
| `/api/products` | GET | 10 минут | Получение списка товаров |
| `/api/products/:id` | GET | 10 минут | Получение товара по ID |


# Практическая работа №22: Балансировка нагрузки

## Описание

Настройка балансировки нагрузки между несколькими backend-серверами. Балансировка позволяет распределять входящий трафик между серверами, обеспечивая отказоустойчивость и масштабируемость приложения.

## Технологии

| Технология | Назначение |
|------------|------------|
| **Nginx** | Основной балансировщик нагрузки |
| **HAProxy** | Альтернативный балансировщик |
| **Node.js + Express** | Backend-серверы |
| **curl** | Тестирование балансировки |

## Алгоритмы балансировки

| Алгоритм | Описание |
|----------|----------|
| **Round Robin** | Запросы распределяются по кругу (по умолчанию в Nginx) |
| **Least Connections** | Запрос направляется на сервер с наименьшим количеством активных соединений |
| **IP Hash** | IP-адрес клиента определяет сервер (сессии закрепляются) |

# Практическая работа №23: Docker + Контейнеризация

## Описание

Контейнеризация приложения с балансировкой нагрузки с использованием Docker. Все компоненты системы (backend-серверы и Nginx) запускаются в отдельных контейнерах, управляемых через Docker Compose.

## Технологии

| Технология | Назначение |
|------------|------------|
| **Docker** | Контейнеризация приложений |
| **Docker Compose** | Оркестрация многоконтейнерных приложений |
| **Nginx** | Балансировщик нагрузки |
| **Node.js + Express** | Backend-серверы |

## Преимущества контейнеризации

| Преимущество | Описание |
|--------------|----------|
| **Портативность** | Приложение работает одинаково на любой машине |
| **Изоляция** | Контейнеры не конфликтуют друг с другом |
| **Легковесность** | Запуск за секунды, меньше ресурсов чем VM |
| **Масштабируемость** | Легко запустить множество копий сервиса |
