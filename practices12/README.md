В ходе выполнения контрольной работы №2, состоящей из практических заданий 7-11, было разработано веб-приложение на Node.js (Express) + React, реализующее интернет-магазин с системой аутентификации, JWT-токенами и ролевой моделью доступа (RBAC).

Были изучены и применены на практике следующие технологии и подходы:

♥ Хеширование паролей с использованием библиотеки bcrypt

♥ JWT-аутентификация (access и refresh токены)

♥ Ролевая модель доступа (RBAC)

♥ Документирование API с помощью Swagger

♥ Работа с токенами на фронтенде и их автоматическое обновление

♥ Полноценный CRUD для товаров и пользователей

♥ Интеграция бэкенда и фронтенда

1 Практическое занятие №7 — Хеширование паролей (bcrypt)
Реализована регистрация и вход с хешированием паролей через bcrypt

2 Практическое занятие №8 — JWT токены
Добавлена генерация access-токена при входе
Создан middleware authMiddleware для защиты маршрутов
Добавлен защищённый маршрут GET /me

3 Практическое занятие №9 — Refresh-токены
Введены refresh-токены для обновления access-токенов
Реализован маршрут POST /refresh для получения новой пары токенов
Настроена ротация refresh-токенов

4 Практическое занятие №10 — Хранение токенов на фронтенде
React-приложение с хранением токенов в localStorage
Добавлены перехватчики axios для автоматической подстановки токена и обработки 401 ошибок

5 Практическое занятие №11 — Управление доступом (RBAC)
В модель пользователя добавлено поле role (user, seller, admin)
Создан middleware roleMiddleware для проверки прав доступа
Реализовано управление пользователями (только для admin)
Настроено отображение кнопок и форм на фронтенде в зависимости от роли
