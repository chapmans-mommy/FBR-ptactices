### Вопросы
## ИСТОРИЯ СОЗДАНИЯ DOCKER
Год                                 	        Событие
2013	                Docker Inc. (основана Соломон Хайкс) выпускает первую версию Docker
До Docker	            Виртуальные машины (VM) — тяжеловесные, долгий запуск, много ресурсов
Проблема	            Приложения работают на одной машине, но ломаются на другой ("it works on my machine")
Решение	                Docker упаковывает приложение со всеми зависимостями в контейнер

Docker появился в 2013 году как решение проблемы "на моей машине работает". До этого использовали виртуальные машины, но они были слишком тяжёлыми. Docker сделал контейнеризацию лёгкой и доступной.

## ЧТО БЫЛО ДО DOCKER?
1. Первым шагом стало появление системного вызова chroot в Unix V7. 
2. Появление Jails во FreeBSD. Они расширили идею chroot, добавив изоляцию не только файловой системы, но и других критически важных      компонентов: собственный hostname, выделенный IP-адрес, изолированную сеть и свою таблицу пользователей.
3. В 2004 году компания Sun Microsystems представила технологию Zones для своей операционной системы Solaris.
4. Появление LXC (Linux Containers) в 2008 году. LXC объединил две ключевые технологии:
namespaces: обеспечивают изоляцию, давая процессу свой собственный вид системы (изолированная сеть, список процессов и т.д.).
cgroups (Control Groups): обеспечивают управление ресурсами, ограничивая количество процессорного времени, памяти и т.д., которое может использовать группа процессов.

## ПОЧЕМУ DOCKER?
Преимущество	                             Объяснение
Легковесность(3)	        Контейнеры делят ядро, запускаются за секунды (VM — минуты)
Портативность(1)	        Запускается одинаково на Linux, Windows, Mac, в облаке
Изоляция(2)	            Каждый контейнер имеет свои зависимости, не конфликтуют
Воспроизводимость	    Dockerfile описывает всё окружение — можно повторить на любой машине
Экосистема	            Docker Hub — тысячи готовых образов (nginx, postgres, redis, node)
Масштабирование(4)        Легко запустить 10 копий одного сервиса через Docker Compose

Docker популярен, потому что он лёгкий (секунды на запуск), портативный (работает везде одинаково), изолирует зависимости и имеет огромную экосистему готовых образов.

Основные альтернативы — Podman (безопаснее, работает без демона), каждый контейнер запускается как дочерний процесс прямо из вашей CLI, и по умолчанию всё работает от имени обычного пользователя (rootless). Сontainerd (стандарт в Kubernetes) Это низкоуровневый компонент, который отвечает только за базовые вещи: скачать образ, создать контейнер, запустить его и следить за процессом. Он высокопроизводительный, стабильный и ничего лишнего не делает. Однако Docker проще в использовании и имеет больше документации.

## ЧТО ТАКОЕ CONTAINER (КОНТЕЙНЕР)
Контейнер — это запущенный экземпляр образа. Изолированная среда с собственными:
Файловой системой
Сетевыми интерфейсами
Процессами
Переменными окружения

Отличия контейнера от виртуальной машины:
Характеристика	            Контейнер Docker	    Виртуальная машина
Запуск	                     Секунды	                  Минуты
Размер	                     Мегабайты	                Гигабайты
Своё ядро	               Нет (делит с хостом)	            Да
Изоляция Процессов	           Полная                   Частичная
Ресурсы	                        Мало	                   Много

кибернетикa:
Контейнер — это изолированная среда для запуска приложения. Он использует ядро хостовой ОС, поэтому лёгкий и быстрый. В отличие от виртуальной машины, у контейнера нет своего ядра.

## ПОЧЕМУ COPY package*.json ./
Сначала копируем только package.json, потому что Docker кэширует слои. Зависимости меняются редко, код — часто. Если сначала скопировать package.json, выполнить npm install, а потом скопировать код, то при изменении кода слой с npm install возьмётся из кэша. Это ускоряет сборку образа.

## ЧТО ТАКОЕ npm install
npm install — это команда Node.js, которая скачивает и устанавливает все зависимости проекта из package.json в папку node_modules.

## ОБРАЗ
Образ - бинарный файл, который содержит всё необходимое для запуска приложения: операционную систему, код, библиотеки, зависимости и настройки. Из одного образа можно создать много контейнеров.
